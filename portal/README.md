# Omnignis Church Portal (v2)

Next.js 14 app for `portal.omnignis.com`. Churches self-register, verify by
6-digit email code, connect their Facebook Page once via OAuth, and get
automated livestream attendance reports by email — generated by a scheduled
Python GitHub Action in this same repo.

Separate repo + separate Vercel project from the static marketing site.

## Structure

```
app/
  page.js                  session check -> /dashboard or /login
  signup/ login/ verify/   auth (Supabase email+password, 6-digit OTP)
  forgot/ reset/           password reset flow
  dashboard/               connection status, connect/disconnect, report summary
  select-page/             page picker (multi-page Facebook accounts)
  account/                 edit details, change password, delete account
  api/facebook/start       creates CSRF state, returns FB OAuth dialog URL
  api/facebook/callback    validates state, exchanges tokens, stores encrypted
  api/facebook/pages       lists granted pages (id+name only — tokens never reach browser)
  api/facebook/select      completes connection with the chosen page
  api/facebook/disconnect  revokes on Facebook + deletes stored tokens
  api/facebook/data-deletion   Meta's Data Deletion Request callback
  api/account/delete       full self-serve account deletion
lib/                       crypto (AES-256-GCM), supabase clients, bearer auth
report/report.py           the scheduled report generator (GitHub Action)
.github/workflows/report.yml   daily cron, 19:00 UTC
schema.sql                 Supabase schema — re-runnable
```

## Setup, in order

### 1. Supabase
1. Run `schema.sql` in the SQL Editor (safe to re-run over the v1 schema).
2. Auth > Providers > Email: make sure **Confirm email** is ON.
3. Auth > Email Templates > **Confirm signup**: the template must contain
   `{{ .Token }}` so the email carries the 6-digit code (the verify page uses
   `verifyOtp`, not the confirmation link).
4. Auth > URL Configuration: set Site URL to `https://portal.omnignis.com` and
   add `https://portal.omnignis.com/reset` to Redirect URLs (password reset).

### 2. Vercel
1. New project from this repo, add the domain `portal.omnignis.com`
   (CNAME in Squarespace DNS -> `cname.vercel-dns.com`).
2. Set every variable from `.env.example` in Project Settings > Environment
   Variables. Generate the encryption key with:
   `node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"`
   **The same key goes into the GitHub Action secrets — reports can't decrypt
   tokens without it.**

### 3. Facebook app (developers.facebook.com)
1. Add product **Facebook Login** > Settings:
   - Valid OAuth Redirect URIs: `https://portal.omnignis.com/api/facebook/callback`
2. App Settings > Basic:
   - Privacy Policy URL: `https://omnignis.com/privacy.html`
   - **Data Deletion Callback URL**: `https://portal.omnignis.com/api/facebook/data-deletion`
3. Permissions used: `pages_show_list`, `pages_read_engagement`, `read_insights`.
   Until App Review passes, only admins/testers of the app can connect —
   the dashboard tells churches this and points them to info@omnignis.com.

### 4. GitHub Action (reports)
Repo Settings > Secrets and variables > Actions, add:
`SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `TOKEN_ENCRYPTION_KEY`,
`FACEBOOK_APP_SECRET`, `RESEND_API_KEY`.
Test with the **Run workflow** button before trusting the cron.

### 5. Resend
Verify the `omnignis.com` sending domain and make sure `reports@omnignis.com`
is allowed as a from address.

## Security model (unchanged from v1, extended)
- Browser uses the anon key; RLS restricts every read to `auth.uid()`.
- All writes to `facebook_connections` and all token handling happen in server
  routes using the service role key.
- Page + user tokens stored AES-256-GCM encrypted (`[12B IV][ct][16B tag]`,
  base64) — byte-compatible between `lib/crypto.js` and `report/report.py`.
- OAuth CSRF states are single-use with a 10-minute TTL, cleared on each new
  connection attempt.
- `appsecret_proof` sent on every Graph API call.
- Disconnect and account deletion revoke the grant on Facebook's side, not
  just locally.
